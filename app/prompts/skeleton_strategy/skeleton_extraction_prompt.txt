--------------------------------------------------
WHAT IS A SKELETON
--------------------------------------------------

A skeleton is a structural map of the receipt items.

For EACH product item, you must identify:
- Its SEQUENCE NUMBER (1-based, ordered)
- A START ANCHOR (text that clearly marks the beginning of the item)
- An END ANCHOR (text that clearly marks the end of the item)

--------------------------------------------------
ITEM BOUNDARY RULE (MANDATORY)
--------------------------------------------------

- In Brazilian NFC-e receipts, EACH product item ends AFTER its COFINS tax block.
- The COFINS section represents the FINAL boundary of an item.
- A new item may ONLY start AFTER the previous item's COFINS block ends.

You MUST use this rule to determine item boundaries.

ANCHOR RULES
--------------------------------------------------

- Anchors MUST be exact text snippets copied from the input text.
- Anchors MUST be:
  - Short

- Anchors MUST NOT be:
  - Entire blocks
  - Numeric-only values
  - Calculated or inferred text

Examples of GOOD anchors:
- First line of the product description
- A unique product header line
- A clear COFINS header or closing line

Return ONLY a valid JSON object.
